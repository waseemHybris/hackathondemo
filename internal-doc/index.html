  • SUB publishProductUpdated

    Operation IDpublishProductUpdated

    Accepts the following message:

    the product updated payloadproductUpdatedEvent

    It includes the product updated changes

    • message
    object
    uid: productUpdatedEventPayload

    The attributes provided in the Product Updated event. The event is generated when one of the following fields is updated:

    • name
    • description
    • type
    • parentId
    • active
    • published
    • media
    • taxCode
    • customAttributes
    • variantAttributes Or when a new association is created or removed between the product and a standing order configuration.
    created
    boolean

    Indicates if the event is a Product Creation event.

    object

    The attributes available when a product is created or updated.

    id
    string

    The imported product's identifier.

    referenceId
    string

    The imported product's unique identifier. When a product is imported, the product ID is the same as the reference ID.

    sku
    string
    [ 1 .. 255 ] characters

    A stock keeping unit (SKU) identification code for the product. This value must be unique by tenant.

    name
    string
    [ 1 .. 255 ] characters

    (Translatable attribute) The product's display name.

    description
    string
    [ 1 .. 4000 ] characters

    (Translatable attribute) The product's description. This value is HTML sanitized by OWASP HTML Sanitizer using HTML common blocks, formatting, links, tables and styles.

    published
    boolean

    True if the product is published in the storefront, otherwise false.

    Default value:
    active
    boolean

    True by default, needs to be set to true to be displayed.

    Default value:
    array<object>
    <= 20 items

    (Translatable attribute) An array of all the media for a given product.

    thumbnail
    string
    [ 1 .. 2000 ] characters

    (Translatable attribute) The thumbnail image URL. There is a url validation applied on this field based on this implementation 'http://docs.oracle.com/javase/7/docs/api/java/net/URL.html', only http and https protocols are supported.

    fullSize
    string
    [ 1 .. 2000 ] characters

    (Translatable attribute) The full-size image URL. There is a url validation applied on this field based on this implementation 'http://docs.oracle.com/javase/7/docs/api/java/net/URL.html', only http and https protocols are supported.

    allowSingleDelivery
    boolean

    Indicates if a single delivery is allowed for products being sold on continuity. Some products are purchasable both once and on continuity, while others are only purchasable on continuity (never once).

    Default value:
    object

    (Translatable attribute) A json object containing specific attributes.
    Requirements:

    • Only a hash map is supported.
    • The hash map keys and values are of type string.
    • Keys cannot be null.
    • Maximum of 200 entries.
    • Maximum map size is 32768 characters.
    When retrieving a product a check is made against the /custom-attributes resource. For custom attribute keys that are set to be 'hidden', the values are not returned unless the request is authenticated with the proper scope.

    Additional properties are allowed.

    array<object>
    <= 10 items

    (Translatable attribute) An array of all the variant dimensions. Dimensions are options for a product, such as size and color.

    key
    string
    [ 1 .. 255 ] characters

    The variant attribute that allows you to group similar variant information together. For example, size or color can be a key type.

    value
    string
    [ 1 .. 255 ] characters

    The key's value.

    swatch
    string
    variantPosition
    string

    Determines the order of variant products when filtering product resources by parent ID. If the variantPosition is not specified, the variant products are sorted alphabetically by product ID. If they are specified, the value must be unique for each variant product within a specific parent product. If there is a mix of specified and unspecified variantPosition values across the variant products of a parent product, products with a variantPosition are sorted first in ascending numerical order and the ones without are sorted after by name alphabetically (ascending), then by product ID. The service converts this string value into a decimal number. The string value must be a positive decimal number, where a value of 0 will always be first in the list. The service supports up to 10 digits before the decimal and up to 50 digits after the decimal.

    parentId
    string
    must match: ^[a-zA-Z0-9_\-\.\(\)~!\*'+ &]+$[ 2 .. 255 ] characters

    The parent product that the variant belongs to.

    type
    string

    The type of product. Creates one of the following type of products:

    • REGULAR - An individual product to be sold as a standalone offer.
    • PARENT - A product that acts as a base product for its variant products. A regular product only becomes a parent product once it has a variant product associated to it.
    • VARIANT - A product that belongs to the parent product. For example, a parent product is a hat and the variant product is a red hat.
    • COLLECTION - A single product that acts as a container for a group of products. For example, a collection product named patio furniture, can have a table, chair, and umbrella assigned to it.
    • UNKNOWN - The default value. For products that are not assigned to a specific type.
    • KIT - A single product that acts as the container for a group of products organized as a kit. Products of this type cannot be changed to another type.

      Allowed values:
    • "REGULAR"
    • "PARENT"
    • "VARIANT"
    • "COLLECTION"
    • "KIT"
    • "UNKNOWN"
    taxCode
    string
    [ 1 .. 255 ] characters

    The product's tax code.

    array<string>
    [ 0 .. 50 ] items

    An array of collection identifiers that indicate which collections the product belongs to, this is not applicable for the COLLECTION, VARIANT and UNKNOWN product types. The collection products in this list must exist.

    Items:
    string
    must match: ^[a-zA-Z0-9_\-\.\(\)~!\*'+ &]+$[ 2 .. 255 ] characters
    maxQtyInOrder
    number
    [ 0.000001 .. 10000 ]

    If provided, this value determines the maximum quantity of this product that can be bought in one order.

    minQtyInOrder
    number
    [ 0.000001 .. 10000 ]

    If provided, this value determines the minimum quantity of this product that can be bought in one order.

    array<object>

    An array of linked attribute set definition (ASD) codes.

    code
    string

    The ASD code identifier.

    array<string>

    The standing order configuration ID associated to the regular/parent product.

    Items:
    string

    Additional properties are allowed.

    Additional properties are allowed.

  • SUB publishPricebookUpdated

    Operation IDPublishPricebookUpdated

    Accepts the following message:

    the pricebook updated payloadpricebookUpdatedEvent

    It includes the pricebook updated changes

    • message
    object
    uid: pricebookUpdatedPayload

    The attributes provided in the Pricebook Updated event. The event is generated when one of the following conditions is met:

    • when a pricebook is created.
    • when at least one attribute of pricebook is updated.
    • when a pricebook status is updated.
    isCreated
    boolean

    Indicates if the event is a Pricebook Creation event.

    object

    The attributes available in this event payload when a pricebook is created or updated.

    id
    string

    pricebook's identifier.

    name
    string

    the name of the pricebook.

    description
    string

    pricebook's description.

    object
    uid: Currency

    The currency object.

    code
    string
    3 characters

    The ISO-4217 format code of the currency.

    name
    string

    (Translatable attribute) The name of the currency.

    symbol
    string

    (Translatable attribute) The graphic symbol of the currency.

    Additional properties are allowed.

    array<string>
    [ 0 .. 50 ] items

    An array of division identifiers that indicate which divisions are assigned to the pricebook. Providing a null value or not providing this field is identical in behavior as providing an empty array.

    Items:
    string
    must match: ^[a-zA-Z0-9]+$[ 1 .. 255 ] characters
    isDefault
    boolean

    true if the pricebook is default, false otherwise.

    object

    state of the pricebook.

    status
    any

    A comma-separated list of pricebook statuses. Possible values are DRAFT, SCHEDULED, PUBLISHED, COMPLETED, or ARCHIVED.

    object
    uid: PublicationInfo

    The publication object containing information about published pricebook.

    object
    uid: PublicationPeriod

    The publication object containing information about when pricebook must be published

    startAt
    string
    format: date-time

    The time at which the pricebook will be scheduled to become active, the date-time string must represent a time in UTC. Providing a null value is identical in behavior as not providing field and the startDate time will not be set.

    Additional properties are allowed.

    publishedAt
    read-only
    string
    format: date-time

    The time at which the pricebook was last set to active.

    unpublishedAt
    read-only
    string
    format: date-time

    The time at which the pricebook was last de-activated.

    Additional properties are allowed.

    Additional properties are allowed.

    Additional properties are allowed.

    Additional properties are allowed.

    Examples

    {
  "isCreated": true,
  "pricebook": {
    "id": "pricebook_id",
    "name": "pricebook's name",
    "description": "pricebook's description",
    "currency": {
      "code": "CAD",
      "name": "Canadian Dollar",
      "symbol": "$"
    },
    "divisions": [
      "divisionOne",
      "divisionTwo"
    ],
    "isDefault": true,
    "stateManagement": {
      "status": null,
      "publicationInfo": {
        "publicationPeriod": {
          "startAt": "2020-07-21T17:32:28Z"
        },
        "publishedAt": "2020-07-21T17:32:28Z",
        "unpublishedAt": "2020-07-21T17:32:28Z"
      }
    }
  }
}
